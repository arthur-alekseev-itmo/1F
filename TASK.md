## 1. Придумать и описать грамматику языка со следующими свойствами

В описании грамматики должны быть

- [ ] неформальное описание
- [ ] приоритет операций
- [ ] допустимые в языке типы данных (продумайте их сами)

Вариант можно выбрать на основе своих вкусовых предпочтений. Подготовьте письменный отчет со всей необходимой информацией.

## Аппрувленное описание языка

Язык общего назначения, функциональный, называется 1Ф
Фактически, просто некое подмножество языка OCaml со схожим но другим синтаксисом

Точно
- [ ] Константы (числа, булы, строки)
- [ ] Объявление функций/значений
- [ ] Вызовы функций
- [ ] If/Then/Else
- [ ] Операторы с различными приоритетами (в том числе свои)

Опционально
- [ ] Объявляние своих типов данных (алгебраические и записи)
- [ ] Инстанциация своих типов данных (конструкторы)
- [ ] Паттерн-матчинг
- [ ] Модули

2. Реализуйте лексический анализ данного языка

- Можно писать на любом языке программирования
- Можно использовать генераторы синтаксических анализаторов
- Решение должно быть оформлено как консольное приложение, принимающее в качестве входа путь к входному файлу
- Результатом лексического анализа является набор токенов
- Каждый* токен должен содержать тип, значение и координаты во входном файле
- Результат лексического анализа должен быть записан в файл с тем же именем, но добавочным расширением .out (если вход -- input.txt, то результат работы записать в input.txt.out).
- Если вход лексически некорректен, сообщить об ошибке с человеческим текстом об ошибке: исключениями в пользователя бросать не стоит.
- [!] Обязательно реализуйте набор тестов для вашего лексера и их запуск в CI/CD gitlab.
- Хороший тест должен читать вход, прогонять его в CI и сравнивать полученный результат с эталонным решением
